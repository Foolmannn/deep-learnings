# Transfer Learning in Detail (with PyTorch)

Transfer Learning is one of the most important techniques in Deep Learning. Instead of training a neural network from scratch, we **reuse a model that has already learned useful features from a large dataset** (usually ImageNet) and adapt it to a new task.

For example:

* Original model: Trained on **1.2 million ImageNet images** with 1000 classes.
* Your task: Classify **cats vs dogs**, **plant diseases**, or **brain tumors**.

Instead of learning edges, shapes, textures, and objects from scratch, the model already knows these features.

---

# Why Transfer Learning?

Training a CNN from scratch requires

* Huge dataset
* Powerful GPUs
* Long training time

Transfer learning solves this by

* Faster convergence
* Better accuracy
* Less data needed
* Lower computation cost

---

# Types of Transfer Learning

There are mainly two methods.

## 1. Feature Extraction

Freeze pretrained layers.

Only train the classifier.

```
Pretrained CNN

[ Frozen Layers ]

↓

New Classifier

↓

Output
```

Advantages

* Fast
* Small datasets
* Prevents overfitting

---

## 2. Fine Tuning

Train some pretrained layers as well.

```
Frozen Layers

↓

Trainable Layers

↓

New Classifier
```

Advantages

* Better accuracy
* Adapts features
* Needs more data

---

# When should we freeze?

Small dataset

```
Freeze everything
Train only classifier
```

Medium dataset

```
Freeze early layers
Train last few layers
```

Large dataset

```
Train almost all layers
```

# Fine Tuning

Instead of freezing everything,

freeze only first layers.

Example

```python
for param in model.parameters():
    param.requires_grad = False
```

Unfreeze layer4

```python
for param in model.layer4.parameters():
    param.requires_grad = True
```

Also classifier

```python
for param in model.fc.parameters():
    param.requires_grad = True
```

Optimizer

```python
optimizer = torch.optim.Adam(
    filter(lambda p: p.requires_grad,
           model.parameters()),
    lr=1e-4
)
```

Notice smaller learning rate.

Fine tuning usually uses

```
1e-4

or

1e-5
```

# Data Transformations

Pretrained models expect ImageNet preprocessing.

```python
from torchvision import transforms

transform = transforms.Compose([
    transforms.Resize((224,224)),
    transforms.ToTensor(),
    transforms.Normalize(
        mean=[0.485,0.456,0.406],
        std=[0.229,0.224,0.225]

```

These mean and std values come from the ImageNet dataset used during pretraining.

---

# Complete Example

```python
import torch
import torch.nn as nn
from torchvision import datasets
from torchvision import transforms
from torchvision import models

transform = transforms.Compose([
    transforms.Resize((224,224)),
    transforms.ToTensor(),
    transforms.Normalize(
        [0.485,0.456,0.406],
        [0.229,0.224,0.225]
    )
])

train_dataset = datasets.ImageFolder(
    "train",
    transform=transform
)

train_loader = torch.utils.data.DataLoader(
    train_dataset,
    batch_size=32,
    shuffle=True
)

weights = models.ResNet18_Weights.DEFAULT
model = models.resnet18(weights=weights)

for param in model.parameters():
    param.requires_grad = False

model.fc = nn.Linear(512,2)

criterion = nn.CrossEntropyLoss()

optimizer = torch.optim.Adam(
    model.fc.parameters(),
    lr=0.001
)

device = torch.device(
    "cuda" if torch.cuda.is_available()
    else "cpu"
)

model.to(device)

for epoch in range(5):

    model.train()

    for images, labels in train_loader:

        images = images.to(device)
        labels = labels.to(device)

        outputs = model(images)

        loss = criterion(outputs, labels)

        optimizer.zero_grad()

        loss.backward()

        optimizer.step()

    print(f"Epoch {epoch+1}: Loss = {loss.item():.4f}")
```

# Feature Extraction vs Fine Tuning

| Feature Extraction                  | Fine Tuning                                                                  |
| ----------------------------------- | ---------------------------------------------------------------------------- |
| Freeze backbone                     | Unfreeze some/all backbone layers                                            |
| Train classifier only               | Train classifier + selected pretrained layers                                |
| Faster                              | Slower                                                                       |
| Less data needed                    | More data needed                                                             |
| Less overfitting                    | Higher risk of overfitting                                                   |
| Higher learning rate for classifier | Lower learning rate for pretrained layers                                    |
| Good for small datasets             | Better when the new dataset is larger or differs significantly from ImageNet |

---

# Best Practices

* Start with a pretrained model instead of training from scratch unless you have a very large dataset.
* Always resize images to the input size expected by the model (commonly 224×224 for many CNNs).
* Use the ImageNet normalization values when using ImageNet-pretrained weights.
* Freeze the backbone first, then fine-tune later if needed.
* Use a smaller learning rate (e.g., `1e-4` or `1e-5`) when fine-tuning pretrained layers.
* Monitor validation accuracy and use early stopping to reduce overfitting.
* Save the best-performing model during training.

---

# Common Mistakes

* Forgetting to replace the final classification layer, causing an output size mismatch.
* Not freezing layers when using a small dataset, which can lead to overfitting.
* Using a learning rate that's too high during fine-tuning, destroying pretrained features.
* Forgetting ImageNet normalization for pretrained models.
* Passing all model parameters to the optimizer when only the classifier should be trained during feature extraction.
* Leaving the model in training mode during evaluation instead of calling `model.eval()`.

---

# When to Use Which Approach

| Dataset Size                                                | Recommended Strategy                                     |
| ----------------------------------------------------------- | -------------------------------------------------------- |
| Less than ~1,000 images                                     | Feature extraction (freeze backbone)                     |
| 1,000–10,000 images                                         | Fine-tune the last one or two blocks plus the classifier |
| More than ~10,000 images                                    | Fine-tune most or all of the network                     |
| Dataset very different from ImageNet (e.g., medical images) | Fine-tune more layers, even with a moderate dataset      |

## Interview / Viva Questions

**1. What is transfer learning?**
Transfer learning is the process of reusing a model pretrained on a large dataset and adapting it to a new but related task, reducing training time and improving performance.

**2. Why is transfer learning useful?**
It requires less data, trains faster, improves accuracy, and leverages features learned from large datasets like ImageNet.

**3. What is the difference between feature extraction and fine-tuning?**
Feature extraction freezes the pretrained backbone and trains only a new classifier. Fine-tuning also updates some or all pretrained layers to better adapt them to the new dataset.

**4. Why do we replace the final layer?**
The original classifier predicts the pretrained dataset's classes (e.g., 1000 ImageNet classes). It must be replaced with a layer that outputs the number of classes in the new task.

**5. What does `requires_grad = False` do?**
It freezes a layer by preventing gradient computation and weight updates during backpropagation.

**6. Why is a smaller learning rate used for fine-tuning?**
A small learning rate preserves useful pretrained features while allowing gradual adaptation to the new task, avoiding large updates that could degrade learned representations.
