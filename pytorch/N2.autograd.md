`autograd` is PyTorch’s **automatic differentiation engine**. It tracks operations on tensors and automatically computes gradients (derivatives) during backpropagation.

If you're learning ML deeply, understanding `autograd` matters because **training neural networks = computing gradients of loss with respect to parameters**.

# 1. Why do we need gradients?

Suppose we have:

[
y = x^2
]

If:

[
x=3
]

Then:

[
y=9
]

Gradient:

[
\frac{dy}{dx}=2x
]

At (x=3):

[
\frac{dy}{dx}=6
]

Instead of manually computing derivatives for huge networks with millions of parameters, PyTorch computes them automatically.

That is `autograd`.

---

# 2. Computational Graph

PyTorch creates a **dynamic computational graph** while operations execute.

Example:

```python
import torch

x = torch.tensor(3.0, requires_grad=True)

y = x**2
z = 2*y + 1
```

Graph:

```plaintext
x ----> y=x² ----> z=2y+1
```

PyTorch stores:

```plaintext
x = 3
y = 9
z = 19
```

and also stores operation history.

---

# 3. `requires_grad=True`

This tells PyTorch:

> Track operations on this tensor.

Example:

```python
x = torch.tensor(2.0,
                 requires_grad=True)

y = x*3

print(y)
```

Output:

```python
tensor(6., grad_fn=<MulBackward0>)
```

Notice:

```python
grad_fn=<MulBackward0>
```

Meaning:

PyTorch remembers multiplication operation.

---

Without:

```python
x = torch.tensor(2.0)

y = x*3

print(y)
```

Output:

```python
tensor(6.)
```

No gradient tracking.

---

# 4. Forward pass

Compute outputs:

```python
x = torch.tensor(3.0,
                 requires_grad=True)

y = x**2

print(y)
```

Output:

```python
tensor(9., grad_fn=<PowBackward0>)
```

Forward:

[
y=x^2
]

---

# 5. Backward pass

Compute derivative:

```python
x = torch.tensor(3.0,
                 requires_grad=True)

y = x**2

y.backward()

print(x.grad)
```

Output:

```python
tensor(6.)
```

Because:

[
\frac{dy}{dx}=2x
]

At:

[
x=3
]

Gradient:

[
6
]

---

## Internally:

Backward traverses graph:

```plaintext
x → y=x²
```

Computes:

[
dy/dx=2x
]

Stores in:

```python
x.grad
```

---

# 6. Chain rule (important)

Neural networks use chain rule repeatedly.

Example:

[
a=x^2
]

[
b=3a
]

[
c=b+2
]

Need:

[
\frac{dc}{dx}
]

Chain rule:

[
\frac{dc}{dx}
=============

\frac{dc}{db}
\times
\frac{db}{da}
\times
\frac{da}{dx}
]

PyTorch computes automatically.

Code:

```python
x = torch.tensor(2.0,
                 requires_grad=True)

a = x**2
b = 3*a
c = b+2

c.backward()

print(x.grad)
```

Compute manually:

[
c=3x^2+2
]

Derivative:

[
6x
]

At x=2:

[
12
]

Output:

```python
tensor(12.)
```

---

# 7. Gradient accumulation

PyTorch **adds gradients** instead of replacing.

Example:

```python
x = torch.tensor(2.0,
                 requires_grad=True)

y = x**2

y.backward()

print(x.grad)
```

Output:

```python
4
```

Run again:

```python
y.backward()

print(x.grad)
```

Output:

```python
8
```

Not:

```plaintext
4
```

Because:

[
4+4=8
]

---

Need:

```python
x.grad.zero_()
```

or optimizer:

```python
optimizer.zero_grad()
```

before next backward.

---

Why?

Training loop:

```python
for epoch in range(epochs):

    optimizer.zero_grad()

    pred = model(X)

    loss = criterion(pred,y)

    loss.backward()

    optimizer.step()
```

# 8. Non-scalar tensors and gradients

Backward works directly only for scalars.

Example:

```python
x = torch.tensor([1.,2.,3.],
                 requires_grad=True)

y = x**2

y.backward()
```

Error:

```plaintext
RuntimeError:
grad can be implicitly created only for scalar outputs
```

Need gradient argument:

```python
y.backward(torch.ones_like(y))
```

Equivalent:

[
\sum y_i
]

Then:

```python
print(x.grad)
```

Output:

```python
tensor([2.,4.,6.])
```

---

# 9. `grad_fn`

Each operation creates a backward function.

Example:

```python
x = torch.tensor(2.,
                 requires_grad=True)

y = x*3

print(y.grad_fn)
```

Output:

```python
<MulBackward0>
```

Examples:

| Operation | grad_fn |
| --------- | ------- |

* | AddBackward

- | MulBackward
  pow | PowBackward
  matmul | MmBackward

---

# 10. Leaf tensors

Leaf tensor:

Created directly by user.

Example:

```python
x = torch.tensor(
    2.,
    requires_grad=True
)
```

Leaf:

```python
True
```

Check:

```python
x.is_leaf
```

Output:

```python
True
```

After operation:

```python
y = x*2

print(y.is_leaf)
```

Output:

```python
False
```

Only leaf tensors store gradients automatically.

# 11. Disable gradient tracking

For inference:

Use:

```python
with torch.no_grad():

    pred = model(X)
```

Benefits:

* Faster
* Less memory
* No graph creation

---

Example:

```python
x = torch.tensor(
    2.,
    requires_grad=True
)

with torch.no_grad():

    y=x*3

print(y.requires_grad)
```

Output:

```python
False
```

---

# 12. `detach()`

Remove tensor from graph.

Example:

```python
x = torch.tensor(
    2.,
    requires_grad=True
)

y = x*3

z = y.detach()

print(z.requires_grad)
```

Output:

```python
False
```

Useful:

Convert tensor → NumPy

```python
tensor.detach().numpy()
```

# 13. Neural network training using autograd

Flow:

```plaintext
Input
 ↓
Model forward pass
 ↓
Prediction
 ↓
Loss
 ↓
backward()
 ↓
Compute gradients
 ↓
optimizer.step()
 ↓
Update weights
```

Code:

```python
model = nn.Linear(1,1)

optimizer = torch.optim.SGD(
        model.parameters(),
        lr=0.01
)

loss_fn = nn.MSELoss()

pred = model(X)

loss = loss_fn(pred,y)

loss.backward()

optimizer.step()

optimizer.zero_grad()
```

# 14. Visualizing autograd graph

Example:

```python
x = torch.tensor(
    2.,
    requires_grad=True
)

y = x**2
z = y*3

z.backward()
```

Graph:

```plaintext
x
│
PowBackward
│
y=x²
│
MulBackward
│
z=3y
```

Backward:

```plaintext
z
↑
y
↑
x
```

Gradients propagate backward.

Hence:

**backpropagation = reverse traversal of computational graph**

# 15. Summary table

| Concept              | Meaning                               |
| -------------------- | ------------------------------------- |
| `requires_grad=True` | Track gradients                       |
| `backward()`         | Compute gradients                     |
| `.grad`              | Stores gradients                      |
| `grad_fn`            | Operation creating tensor             |
| `zero_grad()`        | Reset gradients                       |
| `detach()`           | Remove from graph                     |
| `torch.no_grad()`    | Disable tracking                      |
| Leaf tensor          | User-created tensor storing gradients |
| Computational graph  | Tracks operations                     |
