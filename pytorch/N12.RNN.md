# Recurrent Neural Networks (RNN) in Detail with PyTorch

Recurrent Neural Networks (RNNs) are neural networks specifically designed to process **sequential data**, where the order of the data matters.

Unlike traditional feedforward neural networks, RNNs have **memory**. They remember information from previous time steps and use it when processing the current input.

Typical applications include:

* Natural Language Processing (NLP)
* Speech Recognition
* Time Series Forecasting
* Music Generation
* Video Analysis
* DNA Sequence Analysis

---

# Why not use a normal Neural Network?

Suppose you want to predict the next word.

Sentence:

> "I love eating _____"

To predict the next word, the model should remember:

* I
* love
* eating

A normal feedforward neural network treats every input independently.

```
Input → Hidden → Output

No memory
```

It has no idea about previous words.

An RNN solves this by passing information from one time step to the next.

---

# Basic Idea

Instead of processing inputs independently,

```
x1
x2
x3
x4
```

the RNN processes them one by one.

```
x1 → h1

x2 + h1 → h2

x3 + h2 → h3

x4 + h3 → h4
```

where

* x = current input
* h = hidden state (memory)

---

# Architecture

```
          h0
           |
           v
x1 ---> [RNN Cell] ---> h1
                      |
                      v
x2 ---> [RNN Cell] ---> h2
                      |
                      v
x3 ---> [RNN Cell] ---> h3
                      |
                      v
x4 ---> [RNN Cell] ---> h4
```

Notice that the **same RNN cell** is reused at every time step.

The weights are shared.

---

# Unrolled RNN

People often draw an RNN as:

```
          h0
           |
           v

+------+    +------+    +------+    +------+
| Cell | -> | Cell | -> | Cell | -> | Cell |
+------+    +------+    +------+    +------+
   ^            ^            ^            ^
   |            |            |            |
  x1           x2           x3           x4
```

These are **not different cells**.

They are the same cell repeated over time.

---

# Hidden State

The hidden state is the memory of the network.

```

```

Example

```

love
deep
learning
```

After reading

```

```

hidden state remembers

```
"I"
```

After reading

```

```

```

```

# Mathematical Formulation

At time step t,

Input:

```
x_t
```

Previous hidden state

```
h_(t−1)
```

Current hidden state

[
h_t=\tanh(W_{xh}x_t+W_{hh}h_{t-1}+b_h)
]

Output

[
y_t=W_{hy}h_t+b_y
]

where

* (W_{xh}): input → hidden weights
* (W_{hh}): hidden → hidden weights (recurrence)
* (W_{hy}): hidden → output weights
* (b_h): hidden bias
* (b_y): output bias

---

# Why tanh?

Usually

```
tanh()
```

or

```
ReLU()
```

is used.

tanh outputs

```
[-1,1]
```

which helps stabilize training.

# Backpropagation Through Time (BPTT)

Training an RNN uses

**Backpropagation Through Time (BPTT)**.

Instead of backpropagating through layers,

we backpropagate through **time**.

```
Forward

A → B → C → D

Backward

D → C → B → A
```

The same weights receive gradients from every time step.

---

# Vanishing Gradient Problem

The biggest weakness of vanilla RNN.

During backpropagation,

gradients become

```
0.9 × 0.9 × 0.9 × ...
```

Eventually

```
≈ 0
```

Earlier words receive almost no learning signal.

Example

```
The movie that I watched yesterday with my friends was ...

excellent
```

The word

```
movie
```

is far away.

The RNN may forget it.

---

# Exploding Gradient

Opposite problem.

Gradients become

```
2 × 2 × 2 × 2 × ...
```

Very large values.

Training becomes unstable.

Usually solved using

Gradient Clipping.

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
```

---

# Limitations of Vanilla RNN

* Cannot remember long sequences
* Suffers from vanishing gradients
* Slow (sequential computation)
* Difficult to parallelize
* Poor long-term memory

These limitations led to **LSTM** and **GRU**.

# Complete Example

```python
import torch
import torch.nn as nn

# Hyperparameters
input_size = 10
hidden_size = 20
num_layers = 2
batch_size = 32
seq_len = 15

# Create RNN
rnn = nn.RNN(
    input_size=input_size,
    hidden_size=hidden_size,
    num_layers=num_layers,
    batch_first=True
)

# Input
x = torch.randn(batch_size, seq_len, input_size)

# Initial hidden state
h0 = torch.zeros(num_layers, batch_size, hidden_size)

# Forward pass
output, hn = rnn(x, h0)

print("Output shape:", output.shape)
print("Hidden shape:", hn.shape)
```

Output:

```
Output shape: torch.Size([32, 15, 20])
Hidden shape: torch.Size([2, 32, 20])
```

# Building an RNN Classifier in PyTorch

A common use case is **many-to-one classification**, where only the final hidden state is used to make a prediction (e.g., sentiment analysis).

```python
import torch
import torch.nn as nn

class RNNClassifier(nn.Module):
    def __init__(self, input_size, hidden_size, num_layers, num_classes):
        super().__init__()

        self.rnn = nn.RNN(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True
        )

        self.fc = nn.Linear(hidden_size, num_classes)

    def forward(self, x):
        batch_size = x.size(0)

        h0 = torch.zeros(
            self.rnn.num_layers,
            batch_size,
            self.rnn.hidden_size,
            device=x.device
        )

        out, _ = self.rnn(x, h0)

        # Take the output from the last time step
        out = out[:, -1, :]

        out = self.fc(out)

        return out

# Example usage
model = RNNClassifier(
    input_size=10,
    hidden_size=64,
    num_layers=2,
    num_classes=5
)

x = torch.randn(32, 15, 10)
logits = model(x)

print(logits.shape)  # torch.Size([32, 5])
```

**Why `out[:, -1, :]`?**
For many-to-one tasks, the last output summarizes the entire sequence, making it suitable for classification.

# Best Practices

* Normalize numerical sequence data before training.
* Use `batch_first=True` for more intuitive tensor shapes.
* Clip gradients to reduce exploding gradients:

  ```python
  torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
  ```
* Use packed sequences (`torch.nn.utils.rnn.pack_padded_sequence`) when batching variable-length sequences.
* Prefer **LSTM** or **GRU** over vanilla RNNs for most real-world problems because they handle long-term dependencies much better.
* Move the initial hidden state to the same device as the input (`device=x.device`) to avoid CPU/GPU mismatch errors.

# RNN vs LSTM vs GRU

| Feature              | RNN                                   | LSTM                                      | GRU                                                            |
| -------------------- | ------------------------------------- | ----------------------------------------- | -------------------------------------------------------------- |
| Architecture         | Simple recurrent cell                 | Memory cell + input, forget, output gates | Simplified gated cell (update & reset gates)                   |
| Long-term memory     | Poor                                  | Excellent                                 | Very good                                                      |
| Vanishing gradient   | Severe                                | Greatly reduced                           | Greatly reduced                                                |
| Number of parameters | Fewest                                | Most                                      | Intermediate                                                   |
| Training speed       | Fastest                               | Slowest                                   | Faster than LSTM                                               |
| Computational cost   | Lowest                                | Highest                                   | Medium                                                         |
| Best use case        | Short sequences, educational purposes | Long sequences, translation, speech       | General sequence modeling with a good speed/accuracy trade-off |

---

# When Should You Use an RNN?

Use a vanilla RNN when:

* You are learning the fundamentals of sequence models.
* Your sequences are short.
* You need a lightweight baseline model.

For most practical applications, **LSTMs** and **GRUs** are preferred because they retain information over much longer sequences and are much more robust during training. Today, many NLP tasks have further shifted toward **Transformer** architectures due to their superior performance and parallel processing capabilities, but understanding RNNs remains essential because they form the foundation of modern sequence modeling.
