## What is Optuna?

**Optuna** is an automatic hyperparameter optimization framework for Python.

Instead of manually trying:

```python
lr = 0.1
lr = 0.01
lr = 0.001
```

or doing an expensive grid search, Optuna intelligently explores the search space and focuses on promising parameter values.

---

## Why Optuna?

### Traditional Grid Search

Suppose:

```python
learning_rate = [0.1, 0.01, 0.001]
batch_size = [32, 64, 128]
dropout = [0.2, 0.5]
```

Total combinations:

```python
3 × 3 × 2 = 18
```

Train 18 models.

---

### If Search Space Grows

```python
lr = 20 choices
batch_size = 10 choices
hidden_units = 15 choices
dropout = 10 choices
```

Total:

```python
20 × 10 × 15 × 10
= 30,000 models
```

Not practical.

---

### Optuna Approach

Optuna:

1. Tries some combinations
2. Learns which regions perform well
3. Explores around those regions
4. Avoids bad regions

Result:

* Faster
* Fewer experiments
* Better parameters

# Core Concepts

Optuna revolves around three concepts:

1. Study
2. Trial
3. Objective Function

---

# 1. Objective Function

This is the function Optuna repeatedly calls.

Example:

```python
def objective(trial):

    lr = trial.suggest_float(
        "lr",
        1e-5,
        1e-1,
        log=True
    )

    return score
```

The goal:

```python
maximize score
```

or

```python
minimize loss
```

---

# 2. Trial

Each experiment is called a Trial.

Example:

Trial 1:

```python
lr = 0.01
batch_size = 32
```

Trial 2:

```python
lr = 0.001
batch_size = 64
```

Trial 3:

```python
lr = 0.0007
batch_size = 128
```

Each trial = one model training run.

---

# 3. Study

A Study manages all trials.

```python
study = optuna.create_study(
    direction="maximize"
)
```

Then:

```python
study.optimize(objective, n_trials=50)
```

This means:

```text
Run objective()
50 times
```

with different hyperparameters.

# Parameter Suggestion Methods

---

## suggest_float()

Continuous values.

```python
lr = trial.suggest_float(
    "lr",
    1e-5,
    1e-1
)
```

Possible values:

```python
0.00032
0.012
0.054
```

---

## Log Scale

Very common for learning rates.

```python
lr = trial.suggest_float(
    "lr",
    1e-5,
    1e-1,
    log=True
)
```

Searches:

```text
0.00001
0.00003
0.0001
0.0003
0.001
...
```

instead of linear spacing.

For neural networks:

```python
log=True
```

is usually preferred.

---


# Complete PyTorch Example

## Neural Network

```python
import torch
import torch.nn as nn

class Net(nn.Module):

    def __init__(self, hidden):
        super().__init__()

        self.fc1 = nn.Linear(784, hidden)
        self.relu = nn.ReLU()
        self.fc2 = nn.Linear(hidden, 10)

    def forward(self, x):

        x = x.view(-1, 784)

        x = self.fc1(x)
        x = self.relu(x)
        x = self.fc2(x)

        return x
```

# Pruning (Huge Feature)

One of Optuna's best features.

Suppose:

```python
50 epochs
```

After 5 epochs, model is clearly bad.

Without pruning:

```text
Train all 50 epochs
Waste time
```

With pruning:

```text
Stop early
Try another trial
```

---

## Reporting Intermediate Results

Inside training loop:

```python
for epoch in range(epochs):

    train(...)

    val_acc = validate(...)

    trial.report(
        val_acc,
        epoch
    )
```

---

## Check Pruning

```python
if trial.should_prune():
    raise optuna.TrialPruned()
```

Complete snippet:

```python
for epoch in range(epochs):

    train()

    val_acc = validate()

    trial.report(
        val_acc,
        epoch
    )

    if trial.should_prune():
        raise optuna.TrialPruned()
```

This can reduce tuning time dramatically.

# Saving Study

Instead of losing results:

```python
study = optuna.create_study(
    direction="maximize",
    storage="sqlite:///optuna.db",
    load_if_exists=True
)
```

Creates:

```text
optuna.db
```

Now results persist.

---

# Resuming Study

Later:

```python
study = optuna.load_study(
    study_name="my_study",
    storage="sqlite:///optuna.db"
)
```

Continue:

```python
study.optimize(
    objective,
    n_trials=100
)
```

# Typical Modern Workflow

```text
Create train/validation split
        ↓
Define objective()
        ↓
Optuna suggests hyperparameters
        ↓
Train model
        ↓
Evaluate validation score
        ↓
Report score to Optuna
        ↓
Prune bad trials
        ↓
Find best parameters
        ↓
Retrain on full training set
        ↓
Evaluate on test set
```

## Best Practices

1. Tune **learning rate first**.
2. Use `log=True` for learning rates and weight decay.
3. Enable pruning for deep learning models.
4. Save studies to SQLite.
5. Use validation loss/accuracy, never test data.
6. Start with 20–50 trials; increase if the search space is large.
7. For modern PyTorch projects, a common setup is:

   * Optimizer: `AdamW`
   * Learning Rate: tune between `1e-5` and `1e-2`
   * Weight Decay: tune between `1e-6` and `1e-2`
   * Optuna Pruning enabled
   * Results logged with [TensorBoard](https://www.tensorflow.org/tensorboard?utm_source=chatgpt.com) or [Weights & Biases](https://wandb.ai?utm_source=chatgpt.com)

The most important thing to understand is that **Optuna does not train the model itself**. It repeatedly calls your `objective(trial)` function, and your code is responsible for creating the model, training it, evaluating it, and returning a score. Optuna's job is to decide *which hyperparameters to try next*.
