# Long Short-Term Memory (LSTM) in Detail with PyTorch

A **Long Short-Term Memory (LSTM)** network is a special type of **Recurrent Neural Network (RNN)** that is designed to **remember information for long periods** and overcome the **vanishing gradient problem** of vanilla RNNs.

LSTMs were introduced in **1997** by researchers Sepp Hochreiter and Jürgen Schmidhuber.

LSTMs are one of the most widely used models for sequential data and have been the backbone of many applications in NLP, speech recognition, and time-series forecasting.

---

# Why Do We Need LSTMs?

Consider the sentence:

> "I grew up in France. I speak fluent ____."

To predict the missing word (**French**), the model must remember that the person grew up in France, even though that information appeared many words earlier.

A vanilla RNN struggles because information from earlier time steps gradually fades due to the **vanishing gradient problem**.

An LSTM solves this by introducing a **memory cell** and **gates** that control what information to:

* Keep
* Forget
* Add
* Output

---

# RNN vs LSTM

## Vanilla RNN

```
x₁ → h₁ → h₂ → h₃ → h₄
```

Only the hidden state (`h`) carries information.

Memory gradually fades.

---

## LSTM

```
            Cell State (Long-term Memory)
──────────────────────────────────────────►

        ┌─────────────┐
xₜ ───► │  LSTM Cell  │ ───► hₜ
hₜ₋₁ ─► │             │
Cₜ₋₁ ─► │             │ ───► Cₜ
        └─────────────┘
```

There are **two states**:

* **Hidden State (hₜ)** → Short-term memory
* **Cell State (Cₜ)** → Long-term memory

The **cell state** acts like a conveyor belt that allows information to flow through many time steps with minimal modification, enabling the network to preserve important information over long sequences.

---

# Components of an LSTM Cell

An LSTM cell contains four main components:

1. Forget Gate
2. Input Gate
3. Candidate Memory
4. Output Gate

```
            Previous Cell State
                  │
                  ▼
          +----------------+
xₜ ─────► |                |
hₜ₋₁ ───► |   LSTM Cell    |
          |                |
          +----------------+
           │          │
           ▼          ▼
        hₜ (output)  Cₜ
```

---

# 1. Forget Gate

### Purpose

Decides what information should be removed from the cell state.

Example:

Sentence:

```
I live in Nepal.

Now I moved to Canada.
```

The model should forget the previous location.

The forget gate outputs values between **0 and 1**.

```
0 → Forget completely

1 → Keep completely

0.5 → Keep half
```

Equation

[
f_t=\sigma(W_f[h_{t-1},x_t]+b_f)
]

where:

* σ = Sigmoid function
* (W_f) = Forget gate weights
* (b_f) = Bias

The sigmoid function outputs values between 0 and 1.

---

# 2. Input Gate

### Purpose

Determines what **new information** should be stored.

Equation

[
i_t=\sigma(W_i[h_{t-1},x_t]+b_i)
]

Again,

```
0 → Ignore

1 → Store
```

---

# 3. Candidate Memory

Creates new candidate values that could be added to the cell state.

Equation

[
\tilde{C}*t=\tanh(W_c[h*{t-1},x_t]+b_c)
]

The `tanh` activation outputs values in the range:

```
[-1,1]
```

---

# Updating the Cell State

The cell state is updated using both the forget gate and the input gate.

[
C_t=f_t*C_{t-1}+i_t*\tilde{C}_t
]

Interpretation:

```
New Memory

=

Old Memory to Keep

+

New Information to Add
```

---

# 4. Output Gate

Determines which part of the cell state should become the hidden state.

Equation

[
o_t=\sigma(W_o[h_{t-1},x_t]+b_o)
]

Hidden state:

[
h_t=o_t*\tanh(C_t)
]

The hidden state is what gets passed to:

* The next LSTM cell
* Any output layer

---

# Complete Data Flow

```
                Old Cell State
                      │
                      ▼
             +----------------+
             |  Forget Gate   |
             +----------------+
                      │
                      ▼
             Remove old memory

                      │
                      ▼

             +----------------+
             |  Input Gate    |
             +----------------+
                      │
                      ▼

              Add new memory

                      │
                      ▼

             Updated Cell State

                      │
                      ▼

             +----------------+
             | Output Gate    |
             +----------------+
                      │
                      ▼

                 Hidden State
```

# Complete Example

```python
import torch
import torch.nn as nn

# Hyperparameters
input_size = 100
hidden_size = 128
num_layers = 2
batch_size = 32
seq_len = 20

# Create LSTM
lstm = nn.LSTM(
    input_size=input_size,
    hidden_size=hidden_size,
    num_layers=num_layers,
    batch_first=True
)

# Input
x = torch.randn(batch_size, seq_len, input_size)

# Initial states
h0 = torch.zeros(num_layers, batch_size, hidden_size)
c0 = torch.zeros(num_layers, batch_size, hidden_size)

# Forward
output, (hn, cn) = lstm(x, (h0, c0))

print(output.shape)
print(hn.shape)
print(cn.shape)
```

Output:

```text
torch.Size([32,20,128])
torch.Size([2,32,128])
torch.Size([2,32,128])
```

# Building an LSTM Classifier

For many-to-one tasks (e.g., sentiment analysis), we often use the final hidden state.

```python
import torch
import torch.nn as nn

class LSTMClassifier(nn.Module):
    def __init__(self, input_size, hidden_size, num_layers, num_classes):
        super().__init__()

        self.hidden_size = hidden_size
        self.num_layers = num_layers

        self.lstm = nn.LSTM(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True
        )

        self.fc = nn.Linear(hidden_size, num_classes)

    def forward(self, x):
        batch_size = x.size(0)

        h0 = torch.zeros(
            self.num_layers,
            batch_size,
            self.hidden_size,
            device=x.device
        )

        c0 = torch.zeros(
            self.num_layers,
            batch_size,
            self.hidden_size,
            device=x.device
        )

        out, (hn, cn) = self.lstm(x, (h0, c0))

        # Last time step output
        out = out[:, -1, :]

        out = self.fc(out)

        return out


model = LSTMClassifier(
    input_size=100,
    hidden_size=128,
    num_layers=2,
    num_classes=5
)

x = torch.randn(32, 20, 100)

logits = model(x)

print(logits.shape)
```

Output:

```text
torch.Size([32,5])
```

# Packed Sequences for Variable-Length Inputs

When sequences have different lengths, padding wastes computation. PyTorch provides utilities to process only the valid time steps.

```python
from torch.nn.utils.rnn import pack_padded_sequence, pad_packed_sequence

packed = pack_padded_sequence(
    x,
    lengths,
    batch_first=True,
    enforce_sorted=False
)

packed_output, (hn, cn) = lstm(packed)

output, _ = pad_packed_sequence(
    packed_output,
    batch_first=True
)
```

This improves efficiency and prevents the model from learning from padding tokens.

# LSTM vs RNN

| Feature                | Vanilla RNN       | LSTM                      |
| ---------------------- | ----------------- | ------------------------- |
| Memory                 | Hidden state only | Hidden state + Cell state |
| Long-term dependencies | Poor              | Excellent                 |
| Vanishing gradient     | Severe            | Greatly reduced           |
| Gates                  | None              | Forget, Input, Output     |
| Number of parameters   | Fewer             | More                      |
| Training speed         | Faster            | Slower                    |
| Memory usage           | Lower             | Higher                    |
| Best for               | Short sequences   | Long sequences            |

---

# LSTM vs GRU

| Feature              | LSTM                                            | GRU                                         |
| -------------------- | ----------------------------------------------- | ------------------------------------------- |
| Gates                | 3 (Forget, Input, Output)                       | 2 (Update, Reset)                           |
| Cell state           | Yes                                             | No (merged with hidden state)               |
| Hidden state         | Yes                                             | Yes                                         |
| Number of parameters | More                                            | Fewer                                       |
| Training speed       | Slower                                          | Faster                                      |
| Memory usage         | Higher                                          | Lower                                       |
| Performance          | Often slightly better on complex long sequences | Often similar with lower computational cost |

# Best Practices

* Use `batch_first=True` for intuitive tensor shapes.
* Initialize hidden and cell states on the same device as the input (`device=x.device`).
* Clip gradients (`clip_grad_norm_`) to avoid exploding gradients.
* Use dropout (`dropout`) when stacking multiple LSTM layers (`num_layers > 1`) to reduce overfitting.
* Use packed sequences for batches containing variable-length inputs.
* Choose an appropriate hidden size (commonly 64, 128, 256, or 512) based on the task and available compute.
* For text data, use an embedding layer (`nn.Embedding`) before the LSTM rather than feeding one-hot vectors directly.
* For classification tasks, use the last output (`out[:, -1, :]`) or the final hidden state (`hn[-1]`) depending on the architecture. For sequence labeling tasks, use the entire output tensor.

---

# When Should You Use an LSTM?

LSTMs are a strong choice when:

* You need to model **long-term dependencies** in sequential data.
* Your dataset is **small to medium-sized**, where Transformer models may overfit.
* You're working on **time-series forecasting**, **speech recognition**, **handwriting recognition**, or **classical NLP tasks**.
* You require a model that can process data sequentially and retain information over many time steps.

Although **Transformer** architectures dominate many modern NLP applications due to their parallelism and performance on large datasets, LSTMs remain highly relevant for time-series problems, resource-constrained environments, and as a foundational concept for understanding sequence modeling.
