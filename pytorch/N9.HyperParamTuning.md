# Hyperparameter Tuning in PyTorch (Detailed Guide)

Hyperparameter tuning is the process of finding the best values for parameters that are **not learned by the model** during training.

Examples:

| Hyperparameter   | Example            |
| ---------------- | ------------------ |
| Learning Rate    | 0.1, 0.01, 0.001   |
| Batch Size       | 16, 32, 64         |
| Number of Epochs | 10, 50, 100        |
| Optimizer        | SGD, Adam, RMSprop |
| Weight Decay     | 0.0001, 0.001      |
| Dropout Rate     | 0.2, 0.5           |
| Number of Layers | 2, 3, 4            |
| Hidden Units     | 64, 128, 256       |

---

# Why Hyperparameter Tuning Matters

Consider a learning rate:

```python
lr = 1
```

Training may diverge.

```python
lr = 0.0000001
```

Training may become extremely slow.

```python
lr = 0.001
```

Training may converge nicely.

The goal is to find values that maximize validation performance.

---

# Typical Workflow

## 1. Split Data

```python
train_dataset
val_dataset
test_dataset
```

### Purpose

* Train Set → Learn parameters
* Validation Set → Tune hyperparameters
* Test Set → Final evaluation

Never tune on the test set.

---

## 2. Define Search Space

Example:

```python
search_space = {
    "lr": [0.1, 0.01, 0.001],
    "batch_size": [32, 64],
    "optimizer": ["SGD", "Adam"]
}
```

Total combinations:

```
3 × 2 × 2 = 12
```

---

## 3. Train for Each Combination

For every configuration:

```python
lr = 0.001
batch_size = 64
optimizer = Adam
```

Train the model.

Evaluate on validation set.

Store results.

---

## 4. Select Best Configuration

Example:

| LR    | Batch Size | Optimizer | Val Accuracy |
| ----- | ---------- | --------- | ------------ |
| 0.1   | 32         | SGD       | 75%          |
| 0.01  | 32         | SGD       | 82%          |
| 0.001 | 64         | Adam      | 91%          |

Choose:

```python
lr = 0.001
batch_size = 64
optimizer = Adam
```

# Grid Search

Tests every possible combination.

Example:

```python
learning_rates = [0.1, 0.01, 0.001]
batch_sizes = [32, 64]
```

```python
for lr in learning_rates:
    for bs in batch_sizes:
        train_model(lr, bs)
```

Tests:

```
(0.1,32)
(0.1,64)
(0.01,32)
(0.01,64)
(0.001,32)
(0.001,64)
```

---

## Example in PyTorch

```python
learning_rates = [0.01, 0.001]
batch_sizes = [32, 64]

best_acc = 0

for lr in learning_rates:
    for batch_size in batch_sizes:

        train_loader = DataLoader(
            train_dataset,
            batch_size=batch_size,
            shuffle=True
        )

        model = Net()

        optimizer = torch.optim.Adam(
            model.parameters(),
            lr=lr
        )

        train(model, train_loader)

        acc = validate(model)

        if acc > best_acc:
            best_acc = acc
            best_params = (lr, batch_size)

print(best_params)
```

# Random Search

Instead of trying all combinations, choose random combinations.

Research has shown random search often performs better than exhaustive grid search for large spaces.

Example:

```python
import random

lr = random.choice([0.1,0.01,0.001])
batch_size = random.choice([32,64,128])
dropout = random.choice([0.2,0.5])
```

Train and evaluate.

Repeat many times.

---

## Why Random Search Works

Suppose:

```python
lr = [100 values]
batch_size = [3 values]
```

Learning rate affects performance much more.

Grid search wastes effort checking all batch sizes repeatedly.

Random search explores more learning rates.

def objective(trial):

    lr = trial.suggest_float(

    batch_size = trial.suggest_categorical(

# Early Stopping

Avoid unnecessary training.

```python
patience = 5
```

If validation loss doesn't improve for 5 epochs:

```python
stop training
```

Example:

```python
if val_loss < best_loss:
    best_loss = val_loss
    counter = 0
else:
    counter += 1

if counter >= patience:
    break
```

# Example Optuna + PyTorch Workflow

```python
import optuna

def objective(trial):

    lr = trial.suggest_float(
        "lr",
        1e-5,
        1e-1,
        log=True
    )

    hidden = trial.suggest_categorical(
        "hidden",
        [64,128,256]
    )

    model = Net(hidden)

    optimizer = torch.optim.Adam(
        model.parameters(),
        lr=lr
    )

    train(model)

    acc = validate(model)

    return acc

study = optuna.create_study(
    direction="maximize"
)

study.optimize(
    objective,
    n_trials=50
)

print(study.best_params)
```

### Key takeaway

In PyTorch, hyperparameter tuning is usually done by:

1. Creating a validation set.
2. Defining a search space.
3. Training multiple models with different hyperparameters.
4. Comparing validation metrics.
5. Using tools like **Optuna** for efficient automated search.

For modern PyTorch projects, **Optuna + TensorBoard/W&B + Early Stopping** is one of the most common and effective combinations.
